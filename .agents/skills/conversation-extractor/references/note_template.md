# 基础实战笔记模板 (Basic Note Template)

此模板用于快速采集对话内容，完整保留实战过程，只添加元数据。

## 适用场景
- 快速记录Bug修复过程
- 记录技术问题的解决方案
- 记录工具发现和配置方法
- 记录认知洞察
- **任何觉得可能有价值的对话**

## 模板规范

```markdown
---
type: 实战沉淀
class: {核心认知 | 战术规范 | 疑难排查}
level:
summary: {如果是 Level S，必须填写一句话精华}
review_freq: {Weekly | Monthly | Quarterly}
tags:
  - {具体技术栈}
status: 待日清
create_date: {YYYY-MM-DD}
source: AI Dialog
---

# {日期}-实战-{任务名称}

## 第1轮对话

**用户**: {用户的第一个问题}

**AI**: {AI的回答...}

---

## 第2轮对话

**用户**: {用户的第二个问题}
_（即：{如需要，添加澄清注释}）_

**AI**: {AI的回答...}
```

**轮次组织说明**：
- 每轮对话用 `## 第X轮对话` 标识
- 轮次之间用 `---` 分隔
- 一轮可能包含同一主题下的多次追问
- 判断"新一轮"：用户切换话题/任务、明显时间间隔、明确说"下一个..."

## 采集原则

### ✅ 必须保留的内容

- **完整对话流**：用户问题 → AI回答 → 追问 → 深入解释的完整链条
- **推理过程**：AI如何一步步得出结论的思考过程
- **生动例子**：类比、比喻、具体案例（如"超市购物"类比）
- **结构化内容**：表格、对比、框架、工作流图示
- **完整代码/配置**：包含上下文的代码块
- **报错信息**：完整的 Error Stack
- **环境参数**：版本号、OS类型、依赖等

### ✨ 可以添加的内容（问题澄清注释）

对于**依赖上下文的简短问题**，添加澄清注释以提升未来可读性：

**格式示例**：
```markdown
**用户**: 那 qa-appender 是哪个阶段的？
_（即：qa-appender 技能是用于日清阶段还是提炼阶段的？）_
```

**添加原则**：
- ✅ 只优化指代不明、缺少上下文的简短问题
- ✅ 斜体 + 括号明确是AI补充的理解
- ✅ 保留原问题，不替换
- ❌ 不要过度解释，只补充关键上下文

详细规范见：[extraction_criteria.md](extraction_criteria.md)

### ❌ 不做的事情

- ❌ **不添加内容结构化标记**（如 ## 问题 / ## 解决方案）
  - ✅ **可以添加轮次标识**（如 ## 第X轮对话）来组织多轮对话
- ❌ **不精简内容**（不删除例子、不压缩推理过程）
- ❌ **不做价值判断**（不评估是否资产）
- ❌ **不重组内容**（保持对话的原始顺序）

### ⚠️ 可以删除的内容（最小化删减）

只删除**明显无关**的内容：
- ✂️ AI的礼貌用语（"好的，我来帮你..."）
- ✂️ 工具调用提示（"让我先读取文件"）
- ✂️ 明显的无效尝试（完全跑偏且无参考价值）

**原则**：有疑虑时，保留！

## 为什么这样设计？

### 精简会丢失

| 丢失的内容 | 后果 |
|-----------|------|
| **生动例子**（超市购物类比） | 失去直觉理解，变成干巴巴的道理 |
| **表格对比**（三阶段工作流） | 看不出具体操作步骤 |
| **推理过程**（为什么分离） | 只知道结论，不知道为什么 |
| **语境**（当时的困惑） | 未来看不懂当时为啥问这个问题 |

### 结构化标记会破坏

- 对话的自然流动和渐进展开
- 问题-追问-解释的有机推理链条
- 例子、类比和推理的上下文关系

## 三阶段流程定位

```
📥 采集阶段（本技能）
   → 完整保留，只加 frontmatter
   → 时间：30秒

📋 日清阶段（人工 or 新技能）
   → 初步精简：删冗余、合并重复
   → 可以添加结构化标记（如果需要）
   → 时间：5-10分钟

💎 提炼阶段（process-doc / qa-appender）
   → 深度提炼：提取核心，建立链接
   → 时间：30-60分钟
```

**关键**：每个阶段只做自己该做的事，不越界！

## 后续深化

### 日清阶段（人工处理）
- 删除明显无价值的内容
- 初步分类（FAQ、决策、临时疑问等）
- 决定是否需要添加结构化标记

### 提炼阶段（使用其他技能）

**如果需要记录完整的排查过程和思考复盘**：
- 使用 `process-doc-generator` skill（生成完整过程文档）

**如果需要对笔记进行问答追加**：
- 使用 `qa-appender` skill（智能问答追加）
